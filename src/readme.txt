心得：
1.扫描的快慢，不在线程的多少，而是在于网卡/网络的速度，网卡的个数。
2.扫描全网的IPv4.
  人们一听到这个句话，第一想到的是个数：2的32次方。要快，快，足够的快。
  当你实际测试的时候，你应该会理性的想到：流量。
  假设每个SYN包的大小是60字节，这需要多大呢？
  2^32 * 60 == 257,698,037,760 BYTE == 251,658,240 KB == 245,042 MB == 239 GB （精确的是 240GB == 4G * 60 BYTE）。
  看到了吧！挺吓人的，你下载一个Windows ISO文件才多大，想想你需要多长时间吧！
  masscan号称的5分钟扫描全网，那他的网络/设备，大概是 245,042 MB / 60 / 5 == 816 MB/S。
  大概是千兆的网卡/路由/交换机等设备/网络了。万兆的设备/网络有见过的没？
  我等草民的网速大概是10MB/S（号称百兆是bit,非byte），其实用于SYN扫描的速度大概是5MB/S。
  所以，以我的网络要扫描全网的IPv4大概需要：245,042 MB / 5MB/S == 49,008 S == 816 M == 13 H。
  除去私有的网络地址（大概六分之一），那最少也得十一二个小时。因为还有回来的包（ACK）呢？
3.以5MB/S的速度扫描一个节点的所有端口（65536），大概不到10秒（平均5秒吧）。
  咱算算这容量，有两种算法。
  一是正向的：65536（2^16） * 60 BYTE == 3,932,160 BYTE == 3,840 KB == 3MB.
  另一种是反向：5MB/S * 5S = 25MB.
4.用号称最快的masscan扫描整个IPv4的仅仅一个端口，大概需要一天的时间。要是所有的端口，需要182年。还是网速不行啊！
5.其实最关心和重要的协议是HTTPS，RDP，SSH，SMTP，DNS，ICMP，ARP等几个。

设计特点：
1.不支持针对单个IPv4的单个端口的扫描。
  这可以通过子网掩码的值为32实现。
  还可以通过telnet等各种工具实现。
2.扫描IPv6的局域网，建议用链路本地地址。
3.扫描IPv6的互联网，建议用全球单播地址。否则，用本地IPv6地址，会返回目标不可达的ICMPv6。
4.扫描IPv4的互联网，对方的MAC建议用网关的MAC。
5.扫描IPv6的互联网，对方的MAC建议用网关的MAC。
6.准确性第一。
  要尽可能的获取到开放的主机。
  速度过快，反而会获取的少，因为网络设备的流量清洗，路由器或网关的丢包。
7.速度第二。
  在保证准确性的前提下提高速度。
